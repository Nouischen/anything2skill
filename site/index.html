🚀 開始前的準備（一次就好）
1 裝好 Claude Code

下載桌面版 App ↗⚠️ 網頁版（雲端）碰不到你的電腦、不能用。

2 電腦要有這些
Node.js、Python 3.10–3.13、ffmpeg、yt-dlp（影片下載用）。這些是給程式用的底層工具——你不用懂、也不用自己裝：缺哪個，安裝時 Claude Code 會幫你裝好（會先問你要不要）。只蒸純文字／逐字稿時，多數用不到。
3 轉錄：只有蒸「影片」時才可能用到
純文字、文章、逐字稿來源完全不需要轉錄；只有蒸影片、且抓不到官方字幕時，才要把聲音轉成文字。不確定你的電腦是哪種？直接開始就好——Claude Code 會自動偵測，真的需要金鑰時才會停下來照下面教你辦。
有 NVIDIA 顯卡（Windows/Linux）或 Apple Silicon Mac（M 系列）→ 完全免費、在本機轉錄，音檔不離開你的電腦。
其他情況（Intel Mac、AMD/Intel 顯卡、沒有顯卡）→ 需要一把付費的 OpenAI API 金鑰（約 US$0.006／分鐘，音檔會上傳 OpenAI）。
👉 沒有支援的顯卡？教你拿 OpenAI 金鑰
1. 到 platform.openai.com/api-keys 建一把（需先綁卡、儲值一點額度）。
2. 把那串 sk-… 交給 Claude Code，說「幫我設定 OPENAI_API_KEY」。
3. 費用參考：一小時的影片大約 US$0.36，做完就不再收。有 NVIDIA／Apple Silicon 的人可略過。
💻 桌機用；請只蒸你信任的來源、你自己合法取得的內容。

開始使用：全程在 Claude Code 裡對話

你不用自己打任何指令、不用懂技術。用平常講話的方式跟 Claude Code 說，它就自己幫你安裝、下載、轉錄、蒸成 skill。過程中會跳出權限確認（裝套件、下載）——按「允許」即可。啟用哪個 skill，永遠要先經過你點頭。

1

叫它安裝（一次就好）

🤖 對 Claude Code 說

在 Claude Code 裡打這句，它會自己下載安裝：

幫我把 https://github.com/Nouischen/anything2skill 裝到 ~/.claude/skills/anything2skill，並把包內 agents/a2s-distiller.md 複製到 ~/.claude/agents/，之後我要用它把內容蒸成 skill
裝好後重開 Claude Code 一次讓技能生效，就一勞永逸。
熟 Claude Code？用 plugin 兩條指令裝（子代理自動載入）
/plugin marketplace add Nouischen/anything2skill
/plugin install anything2skill@anything2skill
裝完一樣重開一次。
2

叫它蒸成 skill

🤖 對 Claude Code 說

把網址換成你的來源，打這句（也可以直接餵它一份逐字稿或筆記檔）：

用 anything2skill 把 https://你的影片或文章網址 蒸成 skill

來源可以是影片、課程、文章、你自己的筆記或逐字稿。它會先做一次「價值預檢」：值得蒸的才蒸——沒框架的純閒聊、和一句話就問得到的通用方法（直接問就好），它都會老實勸退你。

3

看過再啟用（鑰匙在你手上）

🖱️ 你來把關

蒸完的 skill 先躺進你的 skill vault（在 Claude Code 的 skills 目錄之外，所以 AI 不會自動拿來用）。它會先給你一張一屏的審核卡（這顆做什麼、步驟、禁區、原話錨點、風險註記，想看全文再展開）；你對某一顆明確說「啟用」，它才把它複製一份進生效目錄（你的正本永遠留在 vault、隨時可回滾），並邀你用真任務試跑一次。沒有你的點頭，任何 skill 都不會生效。之後隨時說一句「我的方法庫」，就能看你擁有哪些方法、要用哪顆、哪顆還沒啟用。為什麼這樣設計 ↓

取內容 → 蒸成草稿 → 貼價值標籤 → 融合入庫（可回滾）→ 審核卡 → 你點頭 → 啟用＋當場試跑一次。每次入庫都會存一個回滾點，你的方法庫永遠可以退回上一步。

它幫你做到什麼（不含「讓 AI 變聰明」——見下方誠實聲明）

我們不宣稱什麼

這是整個工具的立場，先講清楚，你再決定要不要用。

隱私：我們只收數字

預設會回傳一筆「匿名使用計數」，幫我們判斷有多少人在用。只收：一組本機隨機碼（無法對應到你）、事件（安裝／完成一次蒸餾）、這次蒸了幾個 skill、來源類型（youtube／文字／課程…）、版本、時間。

絕對不收：你蒸的內容、標題、網址、任何個資或檔案。程式碼開源可自查。關閉：跟 Claude Code 說「幫我把 anything2skill 的匿名回報關掉」（它會在技能包目錄寫一個 config.json 把回報關掉——持久生效、跨 session 都關）。老實說清楚：本包出廠已預填作者的統計端點（只收上述計數），所以預設會回報——關閉就是上面那一句話的事。

🎙️ 關於轉錄：蒸影片時，有 NVIDIA 顯卡或 Apple Silicon Mac 就在本機轉錄、音檔不離開你的電腦；其他情況用 OpenAI API 模式時，音檔會上傳給 OpenAI 轉錄（適用 OpenAI 的 API 資料政策），作者端一樣收不到任何東西。純文字／逐字稿來源不需轉錄。

🔒 蒸出來的 skill 會不會被惡意來源動手腳？（重要，點開看）
先把界線說清楚，因為這關係到你的安全：

· 蒸出來的 skill 是「可執行的 AI 指令」，內容來自你餵的來源。這代表一個惡意來源有可能把壞指令偽裝成一條看起來正常的方法規則，而工具沒辦法保證能把它和真正的方法分開。
· 所以本工具的設計是「信任由你賦予，不由工具自動賦予」：新蒸的 skill 一律先躺進 vault（在 skills 目錄之外，AI 不會自動發現、觸發），而且帶 disable-model-invocation: true。啟用＝你看過審核卡（或全文）、對那一顆明確點頭，agent 才把它「複製一份進生效目錄＋移除不觸發設定」——vault 永遠留著正本（所以每步可回滾、刪掉生效那份也不會遺失方法），沒有你的確認它不會啟用任何東西（想要最嚴格？選配 hook 可改成複製進生效目錄一律親手做）。
· 任何要改動你既有 skill 的更新，都會先出一份預覽（PENDING-UPDATE），你點頭才套用，不會偷改你信任的東西。
· 選配還附一支加固 hook，讓工具自家蒸出的 skill 不會被 agent 自己裝進生效目錄（safe-by-default 防呆）。

但我們必須誠實告訴你這支 hook 的界線：它不是一道能擋住惡意來源的牆。一個被惡意來源當場挾持的 agent，可以繞過它（例如改用別的方式寫檔）——這點我們自己紅隊測過四輪、確認擋不住。防惡意來源真正、也唯一的防線，是你只蒸自己信任的來源＋啟用前自己看一眼。我們不會、也請你不要相信任何「蒸任何來源都安全」的說法。

一句話：工具幫你產出方法庫，但啟用的鑰匙一直在你手上。
運作原理（想深入了解再看）

它不是死板的模板，而是讓你的 Claude Code 讀懂內容、照蒸餾規格把方法抽出來。整條流程：

取內容→Sonnet 子代理讀長逐字稿起草→主模型貼價值標籤→融合裁決→入庫（git 可回滾）→列待審清單

幾個關鍵：最花 token 的「重讀幾萬字逐字稿」派給便宜的 Sonnet 子代理，你的主模型只做小 token、高判斷的融合裁決（所以建議主模型用 Opus 以上求穩，設 Sonnet 也能跑、只是稍打折）；查證模組尚未上線，所以知識型主張一律標 unverified、不會自稱已查證；蒸完只是產出，生效與否你說了算。

適合蒸什麼？不適合蒸什麼？

一句話判斷：這內容有沒有一套你想讓 AI「每次照做」的方法，而且這方法「不是一句話就問得到」的？兩個都 yes 才值得蒸（另一種也值得：含 AI 不知道的獨門事實／數據＝新知識型）；只想留起來讀 → 用 course2notes 做筆記。
